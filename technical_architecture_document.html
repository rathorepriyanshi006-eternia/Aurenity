D3 — Technical Architecture Document

Executive Summary

Aurenity X is not a traditional passive Application Programming Interface (API) scanner. It represents a paradigm shift in SecOps: a self-healing, self-reasoning Autonomous Cognitive API Immune System. By shifting API security from reactive, stateless checking to persistent, graph-augmented, multi-agent cognitive understanding, Aurenity X maps, simulates, and defends modern enterprise API environments in real-time.

2. Technical Explanation

2.1 Discovery & ML Pipelines

Aurenity X ingests codebases directly from repository endpoints. Raw static files parse through the GitHub Code Analyzer and Grok Repository Intelligence to construct endpoint schemas, extraction variables, and configurations.

During discovery, three key artifacts are extracted:

  • API: Represents route, protocol, file source, handler, and parameters.
  • API_DOC: Tracks presence of OpenAPI inline/external documents or validation comments.
  • API_LASTDATE: Tracks the timestamp of the last codebase modifier change or request activity.

These outputs enter the ML Intelligence Layer. Features are constructed (days_since_last_use, documentation_exists) and passed to our Modular Classification engines:

  • Zombie API Classifier: Labels endpoints where days_since_last_use > 50, representing neglected technical debt.
  • Shadow API Classifier: Labels active endpoints (days_since_last_use ≤ 50) lacking verified documentation/owners.
  • Documentation Classification: Labels endpoints as Documented or Undocumented.

The model architecture is modularly swappable (Rule-Based → Random Forest / Gradient Boosting ensemble → XGBoost / Isolation Forest) without changing the pipeline stages.

2.2 Graph & Digital Twin Engine

Rather than presenting static list-based inventories, Aurenity X resolves relationships. Endpoint metadata builds a property graph mapped into Neo4j.

  • Relationship Resolution Engine: Calculates logical path connections (e.g. Client → Gateway → Auth Filter → Microservice → Database).
  • Blast Radius Engine: Calculates recursive dependencies to simulate loss vectors and downstream system bottlenecks.
  • Digital Twin Sandbox: Uses React state systems on the frontend to clone graph topologies, enabling operators to edit paths, sever links, load-test traffic surges, and inject attack vectors in an isolated sandbox overlay.

2.3 Generative AI & Persistent Memory

Standard RAG frameworks fail at security codebase reasoning because they lack structural context. Aurenity X utilizes a GraphRAG architecture:

  • Encodings translate code chunks into vector space in Qdrant.
  • Queries retrieve semantic context from Qdrant combined with topological context (nodes and edges) from Neo4j.
  • GPT-4o acts as the core reasoning engine, guided by LangGraph (multi-step workflow logic) and CrewAI (autonomous agents).
  • Mem0 provides persistent user/infrastructure memory. The Copilot remembers prior remediation steps, database configurations, and environment fixes across scan iterations.

5. Generative AI Architecture

Aurenity X's Cognitive Layer relies on a GraphRAG pipeline connected to OpenAI's GPT-4o. Rather than querying the LLM directly with plain repository code, the request passes through a multi-tiered context parser:

GitHub Repository Code ↓ Code Extraction & Metadata Generation ↓ Topological Relationship Mapping (Neo4j Graph) ↓ Vector Embedding Generation (Qdrant) ↓ GraphRAG Hybrid Context Retrieval (Graph + Vector) ↓ System Prompt Formulation (Structured System, Agent, & Graph Contexts) ↓ GPT-4o Reasoning Engine ↓ Dynamic Output Generation (Risk Explanations & Remediation Scripts)

Prompting Strategy

  • Structured System Prompting: Constrains output variables to JSON parameters or explicit cybersecurity playbooks.
  • Role-Based Agent Prompting: Instantiates dedicated personas (e.g., Red Team Intrusion Agent, Blue Team Defense Analyst) inside CrewAI.
  • Context-Aware Infrastructure Prompting: Hydrates the LLM prompt template with live JSON schemas of the scanned API endpoints, database engines, and security scores.
  • Graph-Augmented Prompting: Appends Neo4j routing paths to context windows, forcing the AI to evaluate risk implications based on actual network connections.

GraphRAG Selection Justification

Standard RAG frameworks retrieve disjointed code chunks. For APIs, this lacks context: a vulnerability in a Gateway controller is irrelevant if the caller cannot query it. By integrating Neo4j into the retrieval loop, GraphRAG queries actual parent-child route structures, ensuring that AI risk explanations and remediation suggestions evaluate the entire transaction journey.

6. Conclusion

Aurenity X combines graph-native intelligence, cognitive AI, infrastructure memory, predictive security analytics, and digital twin technology to transform API security from reactive monitoring into autonomous infrastructure cognition.
